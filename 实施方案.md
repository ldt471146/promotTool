> 文档定位（更新于 2026-03-05）

## 一、项目概述

### 核心价值
将中文用户的模糊需求转化为结构化、高质量的 AI 提示词。

### 技术选型
- **桌面框架**：Tauri 2.x（Rust 后端 + 轻量级）
- **前端**：Vue 3 + TypeScript + Vite + Tailwind CSS
- **数据库**：SQLite（本地存储）
- **AI SDK**：OpenAI/Anthropic/Google AI

### 核心功能
1. 提示词优化引擎
2. 模板库（内置 + 自定义）
3. 历史记录管理
4. 多轮对话优化

## 三、核心模块设计

### 3.1 AI 集成模块

**文件**：`src-tauri/src/commands/ai.rs`

**核心功能**：
```rust
// 优化提示词
async fn optimize_prompt(
    input: String,
    context: Vec<Message>,
    provider: AIProvider
) -> Result<String>

// 多轮对话
async fn continue_conversation(
    session_id: String,
    message: String
) -> Result<String>

// 切换 AI 提供商
enum AIProvider {
    OpenAI,
    Claude,
    Gemini,
}
```

**System Prompt 设计**：
```
你是专业的 AI 提示词工程师。任务：
1. 分析用户模糊需求
2. 通过反问澄清关键信息
3. 生成结构化提示词

输出格式：
- 需求不明确 → 列出 3-5 个澄清问题
- 需求清晰 → 直接输出优化后的提示词

优化原则：
- 明确角色定位
- 具体化任务目标
- 添加输出格式要求
- 包含约束条件
- 提供示例
```

---

### 3.2 数据库模块

**文件**：`src-tauri/src/commands/db.rs`

**表结构**：
```sql
-- 历史记录
CREATE TABLE history (
    id TEXT PRIMARY KEY,
    original_input TEXT NOT NULL,
    optimized_prompt TEXT NOT NULL,
    template_id TEXT,
    tags TEXT,
    created_at INTEGER NOT NULL
);

-- 模板库
CREATE TABLE templates (
    id TEXT PRIMARY KEY,
    name TEXT NOT NULL,
    category TEXT NOT NULL,
    template TEXT NOT NULL,
    variables TEXT,
    created_at INTEGER NOT NULL
);

-- 对话会话
CREATE TABLE conversations (
    id TEXT PRIMARY KEY,
    messages TEXT NOT NULL,
    created_at INTEGER NOT NULL,
    updated_at INTEGER NOT NULL
);
```

**核心方法**：
- `save_history()` - 保存历史
- `get_history(limit, offset)` - 分页获取
- `search_history(keyword)` - 全文搜索
- `get_templates()` - 获取模板
- `save_template()` - 保存模板

## 九、技术难点与解决方案

### 9.1 AI API 稳定性
**问题**：网络波动、限流、超时
**解决**：
- 指数退避重试
- 请求队列
- 离线模式（本地模型备选）

### 9.2 上下文管理
**问题**：Token 超限
**解决**：
- 保留最近 5 轮
- 摘要压缩
- 手动清除

### 9.3 跨平台打包
**问题**：兼容性
**解决**：
- Tauri 自动更新
- GitHub Releases 分发
- 平台特定说明

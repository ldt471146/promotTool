> 基于 IBM、OpenAI 2026 年最佳实践

## 🎯 通用模板

### 基础结构
```
[角色定位]
你是 [专业身份]，拥有 [背景经验]。

[任务描述]
请 [具体动作] [目标对象]，实现 [预期结果]。

[上下文信息]
背景：[相关背景]
约束：[限制条件]

[输出要求]
格式：[Markdown/JSON/代码]
长度：[字数/行数]
风格：[正式/技术/通俗]
```

## 📝 场景模板

### 1. 代码开发

```
你是资深 [语言] 工程师，擅长 [领域]。

请实现 [功能描述]。

技术栈：
- 框架：[框架名]
- 版本：[版本号]

要求：
- [性能要求]
- [安全要求]
- [代码规范]

输出：
1. 实现代码
2. 关键说明
3. 使用示例
```

**示例**：
```
你是资深 Python 工程师，擅长数据处理。

请实现一个 CSV 文件解析器。

技术栈：
- Python 3.10+
- 标准库优先

要求：
- 支持大文件（>1GB）
- 内存占用 <100MB
- 错误行跳过并记录

输出：
1. 完整代码
2. 性能说明
3. 调用示例
```

### 2. 内容创作

```
你是专业 [内容类型] 创作者。

创作关于 [主题] 的 [内容形式]。

目标受众：[读者画像]

核心要点：
- [要点1]
- [要点2]

风格：[语气/长度/结构]

禁止：[避免内容]
```

**示例**：
```
你是专业技术博客作者。

创作关于 Docker 容器化的入门教程。

目标受众：有基础编程经验的开发者

核心要点：
- 什么是容器
- 为什么用 Docker
- 基础命令实战

风格：通俗易懂/2000字/分步教学

禁止：过度技术术语、假设读者已懂 Linux
```

## 🔥 高级技巧

### 技巧 1：思维链引导

```
问题：[复杂问题]

请一步步思考：
1. 理解问题：[问什么]
2. 分析条件：[已知什么]
3. 推理过程：[如何推导]
4. 得出结论：[答案是什么]
5. 验证结果：[是否合理]
```

---

### 技巧 2：多方案对比

```
问题：[决策问题]

请生成 3 个方案：
方案A：[角度1]
方案B：[角度2]
方案C：[角度3]

对比维度：
- 成本
- 时间
- 风险
- 可维护性

推荐：[最佳方案] 因为 [理由]
```

- [要求3]

## 🚫 反模式

### ❌ 过于简单
"写代码"

### ✅ 结构完整
"你是 Python 专家。写一个读取 CSV 的函数，要求：支持大文件、内存<100MB、错误跳过。输出：代码+说明+示例"

---

### ❌ 缺少上下文
"优化这个"

### ✅ 上下文完整
"这是 Spring Boot 订单系统，高并发下库存超卖。技术栈：Redis+MySQL，QPS 5000。请用分布式锁优化"

---

### ❌ 没有示例
"按某种格式"

### ✅ 提供示例
"按以下格式：
输入：蓝牙耳机，续航8小时
输出：🎧 超长续航8小时！
现在转换：[你的输入]"

---

## 💡 快速参考

### 引导思考
- "让我们一步步思考"
- "请详细说明推理过程"

### 提高准确性
- "基于以下信息"
- "如果不确定，请说明"
- "引用具体段落"

### 控制输出
- "不超过 [N] 字"
- "使用 [格式] 格式"
- "分为 [N] 个部分"

### 迭代优化
- "检查并改进"
- "生成 3 个版本"
- "对比优缺点"

---

## 📚 使用流程

1. **选择模板**：根据任务类型选择
2. **填充内容**：替换占位符
3. **添加细节**：补充具体要求
4. **检查清单**：确认完整性
5. **测试迭代**：根据结果优化

---

**核心原则**：清晰 > 简洁，具体 > 抽象，示例 > 描述

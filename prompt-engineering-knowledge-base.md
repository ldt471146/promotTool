## 📌 什么是提示词工程？

**提示词工程是新时代的编程**。它是通过精心设计输入文本（提示词），引导大语言模型（LLM）生成高质量、准确、相关输出的技术和方法论。

**核心价值**：
- 在 AI 驱动的世界中，用自然语言与机器沟通已成为关键技能
- 好的提示词能让模型性能提升 60-85%
- 掌握提示词工程 = 掌握 AI 时代的核心生产力

**应用场景**：
- 构建智能聊天机器人
- 自动化工作流程
- 创意内容生成
- 代码开发辅助
- 数据分析与洞察

## 🎯 核心技术体系

### 1. 零样本提示（Zero-shot Prompting）

**定义**：不提供任何示例，直接描述任务

**适用场景**：简单、通用的任务

**示例**：
```
将以下文本翻译成英文：
"提示词工程是一门重要技能"
```

**优势**：快速、简洁
**劣势**：对复杂任务效果有限

---

### 2. 少样本提示（Few-shot Prompting）

**定义**：提供 1-5 个示例，让模型学习模式

**适用场景**：需要特定格式或风格的任务

**示例**：
```
将产品描述转换为营销文案：

示例 1：
输入：蓝牙耳机，续航 8 小时
输出：🎧 超长续航 8 小时，音乐不间断！

示例 2：
输入：智能手表，心率监测
输出：❤️ 实时心率监测，守护您的健康！

现在请转换：
输入：无线充电器，快充 20W
```

**关键要素**：
- 示例要有代表性
- 格式要一致
- 通常 2-3 个示例最佳

---

### 3. 思维链（Chain-of-Thought, CoT）

**定义**：要求模型一步步思考，展示推理过程

**适用场景**：复杂推理、数学计算、多步骤任务

**关键词**：
- "让我们一步步思考"
- "请详细说明推理过程"
- "分步骤解决"

**示例**：
```
问题：一个班级有 30 名学生，60% 是女生，女生中 40% 戴眼镜。
请问有多少女生戴眼镜？

请一步步思考并给出答案。
```

**效果提升**：在复杂推理任务上可提升 40-60% 准确率

---

### 4. 思维树（Tree of Thoughts, ToT）

**定义**：探索多条推理路径，选择最优解

**适用场景**：需要探索多种可能性的复杂问题

**示例**：
```
问题：如何优化这个系统的性能？

请生成 3 种不同的优化方案：
方案 1：[从缓存角度]
方案 2：[从数据库角度]
方案 3：[从架构角度]

然后评估每个方案的优缺点，推荐最佳方案。
```

---

### 5. 提示链（Prompt Chaining）

**定义**：将复杂任务拆解为多个子任务，串联执行

**适用场景**：多步骤、需要中间结果的任务

**示例**：
```
任务：分析这份 50 页的报告

第一步：总结每个章节的核心观点
第二步：提取关键数据和结论
第三步：生成 500 字的执行摘要
```

**优势**：
- 提高准确性
- 便于调试
- 可复用中间结果

---

### 6. 检索增强生成（RAG）

**定义**：结合外部知识库，提供准确信息

**适用场景**：需要最新信息或专业知识的任务

**示例**：
```
基于以下文档回答问题：

文档：
[粘贴相关文档内容]

问题：根据文档，如何配置 Redis 集群？

要求：
- 只使用文档中的信息
- 如果文档中没有，明确说明"文档中未提及"
```

**核心价值**：解决 LLM 的知识截止日期问题

---

### 7. 元提示（Meta-prompting）

**定义**：让 AI 帮你优化提示词

**示例**：
```
我想让 AI 帮我写一篇技术博客。
请帮我设计一个高质量的提示词，包含：
- 角色定位
- 具体要求
- 输出格式
- 约束条件
```

---

### 8. 角色提示（Role Prompting）

**定义**：为 AI 分配专业角色和背景

**示例**：
```
你是一位有 10 年经验的 Java 架构师，擅长高并发系统设计。
请设计一个秒杀系统，预计 QPS 10 万。
```

**效果**：让输出更专业、更符合领域规范

## ✅ OpenAI 官方六大策略

### 策略 1：写清楚指令

**技巧**：
- 包含细节以获得更相关的答案
- 要求模型扮演角色
- 使用分隔符（###、"""）分隔不同部分
- 指定完成任务所需的步骤
- 提供示例
- 指定输出长度

### 策略 2：提供参考文本

**技巧**：
- 指示模型使用参考文本回答
- 要求引用参考文本

### 策略 3：拆分复杂任务

**技巧**：
- 将大任务分解为小任务
- 使用意图分类
- 总结或过滤长对话
- 分段处理长文档

### 策略 4：给模型时间思考

**技巧**：
- 要求模型先制定解决方案
- 使用内心独白
- 询问是否遗漏内容

### 策略 5：使用外部工具

**技巧**：
- 使用搜索实现知识检索
- 使用代码执行进行计算
- 调用外部 API

### 策略 6：系统地测试变更

**技巧**：
- 在标准答案基础上评估
- 持续迭代优化

## 🚫 常见错误与避免方法

### 错误 1：模糊不清
❌ "写一篇关于营销的文章"
✅ "为小型餐厅撰写 500 字的 Instagram 营销指南，重点介绍美食摄影和标签策略"

### 错误 2：缺少上下文
❌ "帮我优化这段代码"
✅ "这是一个 Spring Boot 订单系统，高并发下会库存超卖。请用分布式锁优化。技术栈：Redis + MySQL，QPS 5000"

### 错误 3：没有示例
❌ "帮我写个工具类"
✅ "写一个 Java 日期工具类，需要：格式化（输入 Date，输出 '2026-03-04'）、计算日期差（输入两个日期，输出天数）"

### 错误 4：不指定格式
❌ "分析这段代码的问题"
✅ "分析代码，按以下格式输出：1. 功能性问题 2. 性能问题 3. 安全问题。每项包含：问题描述、影响、修复建议"

### 错误 5：一次要求太多
❌ "设计一个完整的电商系统"
✅ 拆分为多个任务：先设计架构 → 再设计数据库 → 然后设计 API → 最后部署方案

## 🔒 安全注意事项

### 提示注入攻击
**风险**：恶意用户通过特殊输入操纵模型行为

**防御措施**：
- 输入验证和过滤
- 使用系统提示词设置边界
- 限制模型权限

### AI 越狱
**风险**：绕过模型的安全限制

**防御措施**：
- 多层防护机制
- 内容审核
- 持续监控

## 📊 提示词成熟度模型

| 层级 | 模式 | 性能提升 | 示例 |
|------|------|----------|------|
| L1 基础指令 | 简单描述 | 10-20% | "写一段代码" |
| L2 结构化 | 角色+任务 | 20-40% | "你是专家，帮我写..." |
| L3 认知增强 | 思维链 | 40-60% | "一步步思考..." |
| L4 智能自适应 | 思维树+迭代 | 60-85% | "探索多种方案..." |
